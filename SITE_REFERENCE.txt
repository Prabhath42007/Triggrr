2. LOCAL DEVELOPMENT SETUP
----------------------------------------------------------------

Step 1: Install Node.js (v18 or later)
  Download from: https://nodejs.org/en/download

Step 2: Clone or download the project
  git clone https://github.com/YOUR_USERNAME/triggrr.git
  cd triggrr

Step 3: Install dependencies
  npm install
  (This reads package.json and installs React, Vite, Tailwind etc.)

Step 4: Set up environment variables
  Copy .env.example to .env
  Fill in the values (see Section 7 below)

Step 5: Run the dev server
  npm run dev
  Opens at: http://localhost:3000

Step 6: Build for production (test before deploying)
  npm run build
  npm run preview
  (Opens a production-like preview at localhost:4173)

3. DEPLOYMENT (VERCEL — FREE)
----------------------------------------------------------------

One-time setup:
  1. Push the project to GitHub (github.com → New repository)
  2. Go to vercel.com → "Add New Project"
  3. Import your GitHub repo
  4. Framework preset: Vite (auto-detected)
  5. Add environment variables (same as your .env file)
  6. Click Deploy

After that:
  Every time you push to GitHub main branch → Vercel auto-deploys
  Takes about 30–60 seconds

Custom domain (when you buy one):
  Vercel dashboard → your project → Settings → Domains
  Add your domain (e.g. triggrr.io)
  Vercel gives you DNS records to add in your domain registrar

After adding domain, find and replace "YOUR_DOMAIN.com" in:
  - index.html (og:url, og:image, canonical, twitter:image, JSON-LD url)
  - src/data/services.js (SITE.domain)
  These are the only two files that need updating.

5. HOW TO ADD A NEW SERVICE
----------------------------------------------------------------

In src/data/services.js, find the SERVICES array.
Scroll to the bottom — there is a commented-out TEMPLATE block.
Copy it, uncomment it, fill in the fields.

Required fields:
  id          → next number after your last service (e.g. 6)
  slug        → lowercase-hyphenated (e.g. 'invoice-automation')
  nicheId     → must match one of the NICHES[].id values
  title       → full service name
  youtubeId   → 11-char ID from YouTube URL, or null if no video yet

That's it. The service will appear automatically on:
  • Home page cards grid
  • Services page (in its niche group)
  • Footer service links
  • Search results

If you're adding a service in a NEW category/niche:
  First add the niche to the NICHES array (instructions in services.js)
  Then add your service with the new nicheId

6. HOW TO ADD A NEW PAGE / ROUTE
----------------------------------------------------------------

Step 1: Create src/pages/NewPage.jsx
  (Copy NotFound.jsx as a starting template)

Step 2: In src/App.jsx, add one lazy import at the top:
  const NewPage = lazy(() => import('@/pages/NewPage'))

Step 3: In App.jsx, add one Route inside the Layout route block:
  <Route path="new-path" element={<NewPage />} />

Step 4: Add a nav link in src/components/Navbar.jsx:
  Find the NAV_LINKS array and add:
  { label: 'New Page', to: '/new-path' }

Step 5: Add a footer link in src/components/Footer.jsx:
  Find COMPANY_LINKS and add:
  { label: 'New Page', to: '/new-path' }

Done. No other files need changing.

7. ENVIRONMENT VARIABLES
----------------------------------------------------------------

These are stored in a .env file (never commit this to GitHub!).
The .env.example file shows the format without real values.

VITE_SCRIPT_URL
  The Google Apps Script web app URL.
  Get it after deploying your Apps Script (see Section 9).
  Used by Contact.jsx to submit the form.

VITE_RECAPTCHA_KEY
  Your reCAPTCHA v3 site key (public key, safe to expose in frontend).
  Get it from: google.com/recaptcha → Admin Console → Create site
  Choose reCAPTCHA v3, add your domain.

In .env file:
  VITE_SCRIPT_URL=https://script.google.com/macros/s/YOUR_ID/exec
  VITE_RECAPTCHA_KEY=6Lc...YOUR_KEY...

In React code, access them as:
  import.meta.env.VITE_SCRIPT_URL
  import.meta.env.VITE_RECAPTCHA_KEY

On Vercel: add these in the Vercel dashboard under
  Project → Settings → Environment Variables

8. HOW TO ADD A YOUTUBE VIDEO
----------------------------------------------------------------

When you upload a new demo video to YouTube:

Step 1: Get the video ID
  YouTube URL: https://youtu.be/kbkyW1aUen8
  Video ID:    kbkyW1aUen8  (everything after youtu.be/)

Step 2: In src/data/services.js, find the service
  Change: youtubeId: null
  To:     youtubeId: 'kbkyW1aUen8'

Step 3: Save and deploy

The VideoEmbed component handles everything else:
  - Shows a clickable thumbnail first (faster page load)
  - Loads the iframe only when user clicks (saves bandwidth)
  - Shows "Coming Soon" placeholder when youtubeId is null
  - Falls back from HD thumbnail to standard quality if HD isn't ready

10. CSP (CONTENT SECURITY POLICY) NOTES
----------------------------------------------------------------

The vercel.json file has strict security headers. If you add
a new third-party service, you may need to update the CSP.

IMMEDIATE ACTION NEEDED:
  Open vercel.json → find Content-Security-Policy header
  Add "https://cdn.jsdelivr.net" to:
    - style-src
    - font-src
  This allows the Tabler Icons web font to load.

Future additions (when you add Batch 2 features):
  Calendly:       add https://calendly.com to frame-src + script-src
  Crisp chat:     add https://client.crisp.chat to script-src + connect-src
  Razorpay:       add https://checkout.razorpay.com to script-src + frame-src
  Supabase:       add your supabase URL to connect-src

12. SCROLL ANIMATIONS — HOW THEY WORK
----------------------------------------------------------------

Any element with class "animate-on-scroll" starts:
  opacity: 0 and translateY: 28px (invisible, slightly below)

When the element enters the viewport, JavaScript adds "in-view":
  opacity: 1 and translateY: 0 (fades in and slides up)

The IntersectionObserver code lives at the top of each page
component (Home.jsx, Services.jsx, etc.) in a useEffect hook.
It fires once per element, then stops watching (performance).

To stagger multiple elements (e.g. cards in a grid), add delay:
  delay-1 = 0.08s  delay-2 = 0.16s  delay-3 = 0.24s
  delay-4 = 0.32s  delay-5 = 0.40s

To make any new element animate on scroll:
  className="animate-on-scroll delay-2"
  No other changes needed.

IMPORTANT: If you create dynamic class names like delay-${i},
Tailwind won't detect them at build time and will remove those
styles. Use a lookup array of literal strings instead:
  const delays = ['delay-1', 'delay-2', 'delay-3', 'delay-4', 'delay-5']
  className={`animate-on-scroll ${delays[i] ?? 'delay-5'}`}

14. KNOWN GOTCHAS & IMPORTANT NOTES
----------------------------------------------------------------

1. postcss.config.js MUST be in the root folder (same level as
   package.json). Tailwind will not compile without it.

2. vercel.json needs cdn.jsdelivr.net added to style-src and
   font-src in the CSP header, or Tabler Icons won't load.

3. Dynamic Tailwind class names (e.g. `text-${color}`) won't
   survive the production build. Always use full literal class
   names in JSX or component files.

4. The .env file must never be committed to GitHub.
   It's in .gitignore automatically. The .env.example file
   (which has no real values) CAN be committed.

5. Google Apps Script URL changes every time you deploy a new
   version. Update VITE_SCRIPT_URL in Vercel env vars each time.

6. YouTube thumbnails: maxresdefault.jpg (HD) may return 404 for
   new or private videos. VideoEmbed auto-falls back to hqdefault.
   If both fail, the skeleton persists. Fix: make video public on YT.

7. The @/ path alias (e.g. import '@/components/Navbar') maps to
   the src/ folder. Set up in vite.config.js. Only works in Vite
   projects — not in raw Node.js scripts.

8. Sticky navbar height is h-16 (64px = 4rem). If you change it,
   update the hero section's min-h-[calc(100vh-4rem)] to match.

9. WhatsApp pre-filled message is in SITE.whatsappUrl in
   services.js. Edit the ?text= parameter to change the message.

10. The section-divider class (in index.css) creates a purple-tinted
    horizontal line. Use between major sections for visual breathing room.
